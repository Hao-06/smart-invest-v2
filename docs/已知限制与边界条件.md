# 已知限制与边界条件

> 本文档列出 **LiveTradingAgentV2** 系统在哪些边界条件下会**优雅降级**而非崩溃，
> 以及目前的工程性限制与建模假设。**透明对待限制本身就是工程严谨性的一部分。**

---

## 一、数据层

### 1.1 数据源不可用时

| 情况 | 系统行为 |
|---|---|
| 东方财富 K 线接口失败 | 自动回退到腾讯财经源（个股与 ETF 均覆盖） |
| 所有行情源都失败 | 该标的当日无价 → V2 推荐里若包含它会自动跳过（`get_price` 返回 `None` 时不入清单） |
| 资金流接口失败 | regime 信号里 `north_flow_*` 字段缺失；规则判断（offline）不依赖该字段，影响为零 |
| 估值因子接口失败 | high_dividend / reversal 策略当日返回空清单；其它策略不受影响 |
| 个股信息接口（ETF）失败 | ETF 名称已内置 15 只行业 + 5 只宽基静态映射，永不发起网络请求 |
| 沪深300 指数接口失败 | regime/基准 自动用 510300 ETF 作代理 |
| 多因子 CSV 接口失败 | `pit_top_n` 返回空 universe → V2 当日降级为「仅 ETF 策略」（2023 区间就是这种情况） |

**关键设计**：单个数据接口失败**绝不导致整体崩溃** —— 影响范围仅限于「该层拿到更少信息」。

### 1.2 网络环境

| 情况 | 系统行为 |
|---|---|
| 本机配置了代理（如 Clash） | 数据层强制 `NO_PROXY=*` 走直连 —— 自动屏蔽境内站点被绕到境外节点的问题 |
| 接口偶发限流（短期空响应） | 会话内负缓存（30 分钟 TTL）记忆失败接口，避免反复重撞 |
| 长时间断网 | 缓存命中的标的仍能跑（Parquet 长历史已固化） |

### 1.3 数据时点

| 情况 | 系统行为 |
|---|---|
| `as_of` 是周末或节假日 | 行情按 `date <= as_of` 截断，自然取最近交易日 |
| `as_of` 在标的上市日之前 | 行情窗口为空，标的被跳过 |
| 当日盘中调用（数据未收盘） | 取最近已收盘的交易日数据，无前视偏差 |
| 标的当日停牌 | 无开盘价 → 该推荐当日不成交（不阻塞主流程） |

### 1.4 多因子 CSV 的数据可靠边界

| 区间 | 数据可用性 | V2 行为 |
|---|---|---|
| 2024-01-02 ~ 2025-08-07 | ✅ 完整覆盖（约 275 只/日） | momentum + high_dividend 全功能运行 |
| 2025-08-08 ~ 2025-12-27 | ⚠️ 稀疏（最少 1 只/日） | `pit_top_n(min_complete=100)` 自动跳过稀疏日；high_dividend 当日为空 |
| 2026-01-02 及之后 | ❌ 无新增 | universe 取 2025-08-07 的 PIT 快照（合法但偏旧，已诚实披露） |
| 2024-01-02 之前 | ❌ 无 | universe 为空 → V2 降级为仅 ETF 策略 |

## 二、LLM 层

### 2.1 V2 的 LLM 使用边界（关键）

V2 把 LLM 的使用**严格限制在「实盘」场景**：

| 模式 | regime / allocator 行为 | 是否调 LLM |
|---|---|---|
| **回测**（`offline=True`，所有 `scripts/test_live_agent_v2.py` 调用默认） | 规则判断 | ❌ 不调 |
| **实盘**（`main.py recommend --use-llm`） | DeepSeek-R1 推理 + 规则保底 | ✅ 调 |

**为什么回测不调 LLM**：
1. R1 的训练数据有截止日 —— 用 R1 判断 2024-09-24 的市场是「**事后诸葛**」
2. LLM 输出有随机性 —— 同一份输入可能得到不同输出，**回测不可复现**
3. 每天 2 次 LLM 调用 × N 天 = 大量 token 成本 + 网络依赖

### 2.2 LLM 调用稳健性（实盘模式）

| 情况 | 系统行为 |
|---|---|
| 单次 API 错误（5xx / 超时） | tenacity 自动重试 3 次（指数退避，2-20s） |
| 重试后仍失败 | regime/allocator 自动回退到规则判断（`_rule_based_fallback`），**不抛异常**主流程不中断 |
| LLM 输出非合法 JSON | `LLMResponse.parse_json` 3 段式容错；仍失败则回退规则 |
| 输出方向词非约定取值 | 归一化兜底 |
| score / confidence 超出范围 | `_clamp` 强制截断 |

### 2.3 R1 推理模型的特殊限制

| 限制 | 系统处理 |
|---|---|
| 不支持 `response_format` JSON 模式 | 自动降级：在 system prompt 末尾追加 JSON 格式要求 + `parse_json` 容错 |
| 不支持 `temperature` / `top_p` | 客户端自动跳过 |
| 响应时间可达 30-60 秒 | 全局超时延长到 180 秒 |
| 偶发返回空 content | `parse_json` 抛 ValueError → 回退规则判断 |

### 2.4 API key 异常

| 情况 | 系统行为 |
|---|---|
| key 未配置 + offline 模式 | ✅ 完全不受影响（不构造 LLM 客户端） |
| key 未配置 + `--use-llm` | `DeepSeekClient.__init__` 抛 `RuntimeError` 提示填 `.env` |
| key 无效 / 余额不足 | OpenAI SDK 抛 `AuthenticationError` → 回退规则判断 |

## 三、回测引擎

### 3.1 V2 回测的核心约束

| 约束 | 实现方式 |
|---|---|
| **PIT universe** | `FactorLoader.pit_top_n(start)` 用回测起点前最近一个**完整**因子日选股池，跳过 CSV 边界稀疏日 |
| **offline 确定性** | regime/allocator 走规则判断，同一份输入必得同一份输出 —— 完全可复现 |
| **100 股整数倍** | volume = `int(目标价值/价格) // 100 * 100`，强制约束 |
| **当日无价不成交** | `get_price(sym, today)` 返回 `None` 时跳过该笔买入 |
| **现金不足按上限截断** | 自动按可用现金算最大可买 volume；不足 100 股则跳过 |

### 3.2 成交价与平台行为假设（**主动披露**）

| 项 | 假设 | 备注 |
|---|---|---|
| 买入成交价 | T 日**开盘价** | 理想化（无滑点） |
| 卖出成交价 | T 日**开盘价** | 理想化 |
| 估值价 | T 日**收盘价** | 标准 |
| **平台平仓规则** | **假设 T+5 自动平仓** | ⚠️ **对平台行为的假设** —— 真实规则未公开（复赛对接技术文档才知），平台可能是 T+1 / T+10 / 永不平仓 |
| 佣金 | 万 2.5（单笔最低 5 元） | 按 A 股实际费率建模，非平台真实收费 |
| 印花税 | 千 0.5（仅卖出） | 同上 |
| 滑点 | **当前不模拟** | 略偏乐观；真实成交有滑点与冲击成本 |
| 部分成交 | 假设全额成交 | A 股大票通常没问题，小盘股可能高估 |

> 这些是回测领域的常规简化。**复赛阶段平台对接后，T+N 规则、滑点、费率等以「智投未来」平台为准**。

### 3.3 决策时点

| 项 | 假设 |
|---|---|
| 决策基准日 | **T-1 收盘后**（用户晚上跑当晚的决策；或开盘前跑当日决策） |
| 输出格式 | 当日 `[]`（空操作）/ `[{symbol, symbol_name, volume}, ...]`（仅买入） |
| 大盘崩盘 fast-path | 沪深300 单日跌 >3% → 直接推荐 ETF 防御，跳过主流程 |

## 四、运行环境

### 4.1 Python 与依赖

| 项 | 要求 |
|---|---|
| Python 版本 | ≥ 3.11（已测：3.13） |
| 操作系统 | macOS / Linux（已测：macOS 26）；Windows 应可运行，未充分测试 |
| 内存 | ≥ 2 GB（连续长回测建议 4 GB） |
| 磁盘 | < 200 MB（含 Parquet 缓存） |

### 4.2 Streamlit 看板

| 情况 | 系统行为 |
|---|---|
| 首次启动需邮箱 | 已通过 `~/.streamlit/credentials.toml` 跳过 |
| 长回测过程 UI「假死」 | 后台同步执行属正常；建议 CLI (`main.py validate`) 跑长区间 |
| 浏览器兼容 | 已测 Chrome / Safari |

> ⚠️ 当前 Streamlit 看板的「单日决策」Tab 是**早期 6 分析师团队架构**的可视化，V2 的等价输出请用 `python main.py recommend --date YYYY-MM-DD`。后续可统一看板到 V2。

### 4.3 字体加载

## 五、复赛阶段待对接事项

如入围复赛、进入「智投未来」平台对接：

| 事项 | 当前状态 | 复赛要做 |

| LLM 推理 | 回测全程 offline | 实盘可启用（`main.py recommend --use-llm`），R1 思维链可作为「可解释」实物证据 |

## 六、未实现的功能（已知简化）

按对作品价值的优先级倒序：

- ❌ **涨跌停 / 一字板模拟** —— A 股的实际成交摩擦
- ❌ **分钟级行情** —— 当前仅日线
- ❌ **滑点 / 冲击成本** —— 回测假设按开盘价完美成交
- ❌ **个股公告** —— `stock_news_em` 已覆盖部分公告类新闻，但独立的「公告」接口未接入
- ❌ **多标的组合优化** —— V2 用「策略权重 × 排名」启发式合并，未做 Markowitz / Black-Litterman
- ❌ **实盘风控警报** —— 命题为模拟竞赛，无实盘风险

## 七、稳定性保障

**怎么证明 V2 系统能跑？** 任一命令成功即可：

```bash
# 1. 单日推荐（10 秒级返回 JSON，offline 模式无需 API key）
python3 main.py recommend --date 2026-05-22

# 2. 短区间回测（约 1 分钟）
python3 main.py validate --start 2026-05-12 --end 2026-05-22

# 3. 完整 6 区间分行情验证（约 10 分钟）
bash scripts/run_v2_regions.sh

# 4. 连续长回测（2024~2026，约 5 分钟）
python3 main.py validate --start 2024-01-02 --end 2026-05-18 \
    --out runs/v2_continuous.json
```

任一命令完整跑出 JSON 输出或回测报告，即证明系统健康。
评委可拿这些命令独立验证作品。
