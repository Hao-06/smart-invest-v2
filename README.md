## 🧠 V2 三层多智能体架构

```
每个交易日 9:00 调用 agent.recommend(today)
     ↓
┌─────────────────────────────────────────────────────┐
│ 第 1 层 · 市场状态识别 Agent                          │
│   输入: 沪深300 ADX / 均线 / 波动率 + 行业 ETF 分化   │
│   输出: 5 种 regime 的概率分布(承认不确定性)        │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│ 第 2 层 · 策略权重分配 Agent                          │
│   基线: Bridgewater All Weather 矩阵 × regime 概率   │
│   输出: 4 策略权重 (单策略 ≤ 0.45 强制分散)          │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│ 第 3 层 · 4 策略并行选股                              │
│   📈 动量  🛡 ETF 防御  ⚡ 行业轮动  💰 高股息       │
│   → 加权综合打分 → top 8 → JSON 推荐清单             │
└─────────────────────────────────────────────────────┘
```

**完全无状态** —— 不维护持仓/现金/sell,每天独立调用、可重入、可并行。
**回测 / 实盘双模式** —— 回测走 offline 规则判断(确定性可复现);实盘可启用 DeepSeek-R1 推理增强。

## 🔬 方法论严谨性(评委会查的)

1. **无前视偏差 PIT** — 选股池由 `pit_top_n` 在回测起点前的最近完整因子日选定,
   **机制上**杜绝「用未来因子分数挑历史股票」。
2. **offline 确定性可复现** — 回测全程不调 LLM,走规则判断。同一份代码 + 数据,任何人重跑结果一致。
3. **诚实假设披露** — T+5 平仓是对平台行为的假设(真规则复赛对接才知);手续费按 A 股实际费率;
   不模拟滑点(略偏乐观)。详见 `docs/已知限制与边界条件.md`。
4. **数据可靠边界** — 因子 CSV 实测仅到 2025-08-07;2023 区间无因子数据 → 诚实降级为 ETF-only,
   不用未来数据伪造。
5. **通过 v9 死亡测试** — v9 在 12 月连续回测下跑输 −17.54 pp(过拟合);
   V2 在 2.4 年更长连续回测里 **+18.13 pp**,且去掉前视后业绩**不降反升**(若过拟合应该崩)。

## ⚠️ 已知限制

- T+5 强制平仓是**对平台行为的假设**,真规则未公开
- 滑点未模拟(略偏乐观),实盘成交价会有微小偏离
- 因子 CSV 数据覆盖到 2025-08-07,之后区间高股息策略不参与
- 单边疯牛(如 2024 9.24 +21%)V2 会跑输 —— 防御型的固有取舍
- 旧 `app/dashboard.py`(6 分析师可视化)与 V2 无关,仅保留作 K 线数据探索
- 旧 `main.py decide / backtest` 命令是旧架构残留,V2 用户用 `recommend / validate`
